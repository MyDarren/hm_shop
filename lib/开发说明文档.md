创建项目-基本目录使用git管理
1、首选创建适配web平台项目
    flutter create --platforms web hm_shop
2、创建基本目录结构
    -lib
        -api            # 存放请求
        -assets         # 存放资源
        -components     # 存放公共组件
        -contants       # 存放常量文件
        -viewmodels     # 存放类型文件
        -pages          # 存放页面
        -routes         # 存放路由配置
        -stores         # 存放全局状态组件
        -utils          # 存放工具类
        -main.dart      # 入口
3、创建本地和远程仓库并提交
    git init                                    # 初始化仓库
    git add .                                   # 提交本地仓库到暂存去
    git commit -m "初始化项目"                    # 提交本地仓库
    git remote add origin master <远程仓库地址>   # 添加远程仓库
    git push -u origin master                   # 推送主分支到远程


运行flutter项目到其他平台
补全其他平台代码，运行flutter create .
注意：flutter create . 只会补全 Android/iOS/Web/Windows/macOS/Linux 等标准平台，不会生成鸿蒙(Ohos)端；鸿蒙需使用鸿蒙版 Flutter SDK（见下方「运行flutter项目到鸿蒙端」）。
若执行后出现 Java 与 Gradle 版本冲突提示：已通过 android/gradle/wrapper/gradle-wrapper.properties 使用 Gradle 8.7（国内镜像）解决；也可用 flutter config --jdk-dir=<Java17路径> 指定 JDK 17。

运行flutter项目到安卓端
设置gradle和maven仓库
1、打开android/gradle/wrapper/gradle-wrapper.properties设置distributionUrl为国内镜像地址
    distributionUrl=https\://mirrors.cloud.tencent.com/gradle/gradle-8.14-all.zip
2、修改gradle的maven仓库镜像地址(android/settings.gradle.kts)
    maven { url = uri("https://maven.aliyun.com/repository/google") }
    maven { url = uri("https://maven.aliyun.com/repository/releases") }
    maven { url = uri("https://maven.aliyun.com/repository/central") }
    maven { url = uri("https://maven.aliyun.com/repository/public") }
    maven { url = uri("https://maven.aliyun.com/repository/gradle-plugin") }
    maven { url = uri("https://maven.aliyun.com/repository/apache-snapshots") }
    maven { url = uri("https://maven.aliyun.com/nexus/content/groups/public/") }
    maven { url = uri("https://jitpack.io") }
3、设置网络权限
    1、给安卓侧添加网络权限（android/app/src/main/AndroidManifest.xml）
        <uses-permission android:name="android.permission.INTERNET" />
4、打包
    flutter build apk --debug # 生成调试版APK（未签名，用于调试）
    flutter build apk --release # 生成发布版APK（需要签名配置）
        

运行flutter项目到鸿蒙端
运行步骤：
1、下载鸿蒙版flutter替换原有版本
    git clone https://gitcode.com/openharmony-tpc/flutter_flutter.git
2、安装DevEcoStudio-安装模拟器
3、配置鸿蒙环境变量
    export TOOL_HOME=/Applications/DevEco-Studio.app/Contents # mac环境
    export DEVECO_SDK_HOME=$TOOL_HOME/sdk # sdk
    export PATH=$TOOL_HOME/tools/ohpm/bin:$PATH # ohpm
    export PATH=$TOOL_HOME/tools/hvigor/bin:$PATH # hvigor
    export PATH=$TOOL_HOME/tools/node/bin:$PATH # node
    export HDC_HOME=$TOOL_HOME/sdk/default/openharmony/toolchains # hdc
4、清理项目-降低版本-安装依赖
    鸿蒙版本flutter相较于最新版落后，需要降低使用版本
    pubspec.yaml中
        environment:
          sdk: ^3.6.1
    改为
        environment:
            sdk: ^3.4.0
    shared_preferences降低为2.2.2
    flutter_lints降低为3.0.0
    使用flutter clean && flutter pub get清理项目并重新安装依赖
5、项目签名运行到模拟器
    补全鸿蒙侧代码 flutter create .
    打开鸿蒙模拟器，在项目中执行命令 flutter run # 选择对应的设备
6、解决兼容性问题
